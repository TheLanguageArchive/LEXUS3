Dear LEXUS user,

The release candidate version of the upcoming LEXUS is now available at http://corpus1.mpi.nl/lex/lexus.

Although the main focus of this version has been a complete rewrite of the server backend, which you should experience by improved performance and stability, the user interface has been improved at several places:
  • View specification: Views now have a unified interface, both the Lexical Entry View and the List View use the same interface. What's more, you can now define additional views, to be used for instance when generating a PDF. Other uses for views will no doubt be implemented.
  • Queries and filters: queries and filters have also been unified, they use the same technology and have the same interface. The usability of the interface for creating queries is much improved.
  • New Search options: There is a new fast search option in the Lexicon Editor. Simply typing text and pressing enter searches the whole lexicon for the text you typed. There's an option to search case sensitive. Search text is also used when a filter is applied, so you can search within the results of a filter.
  • PDF export: in the lexicon editor there's an "Export to PDF" option. It uses a view (other than the Lexical Entry View and the List View) to generate presentations of lexical entries. The overall output is not very customizable yet, but we welcome your input.
  • Toolbox conversion: in the Workspace there's a menu option "Convert Toolbox files to Lexus" that let's you upload a Toolbox .typ and a .txt file and gives you a Lexus XML file back. You can then import this Lexus XML file into Lexus, or do your own processing on it to enhance the lexicon.
  • General bugfixes: there have been numerous bugfixes to improve the stability and usability of the Lexus interface.
Please help us validating you eventual LEXUS 2 data in this new version! We very much welcome any feedback like problems with your data, bug reports and/or feature request. Please contact us by sending an email to lexus@mpi.nl.
During the validation phase we will be updating this release candidate version frequently to incorporate fixes for all the bugs that you found.
Notice that this release candidate version of LEXUS should not be used for any actual work on your lexica, i.e., work that you would like to persist. The database is scheduled to be reseted in the end of the validation phase but might need to be refreshed at any time! The period for you to validate you data and will be limited to 1 month, so LEXUS 3 improvements will soon be available for your production work.

Regards,
The LEXUS team

NOTE: At this moment ViCoS and the XML import facility available on LEXUS 2 are not yet available in the release candidate version, this will be added in upcoming versions.
